# 多Agent反向钻探工作流配置

## 四Agent角色定义

| Agent | 角色 | 核心能力 | 输入 | 输出 |
|-------|------|----------|------|------|
| **A** | BOM拆解员 | 产品结构分析、技术路线判断 | 产品名 | BOM清单 |
| **B** | 市场集中度审计员 | 供应商格局、市占率追踪 | BOM清单 | 供应商排名表 |
| **C** | 财务定价权评估员 | 财报分析、定价权判断 | 供应商名单 | 财务评级 |
| **D** | 卡脖子综合评级员 | 图论量化、投资建议 | A+B+C结果 | 最终报告 |

## Agent A: BOM拆解员

### 触发条件
- 用户指定了终端产品
- 用户说"拆解XX的产业链"

### 执行流程
```
1. [搜索] "[产品名] BOM/teardown/cost breakdown"
2. [搜索] "[产品名] 核心零部件 供应商"
3. [搜索] "[产品名] 关键原材料 技术路线"
4. [过滤] 去掉标准化零部件（电容、电阻、PCB等）
5. [排序] 按成本占比和技术壁垒排序
6. [输出] BOM清单 → 输入给Agent B
```

### 关键提示词
```
你是半导体/硬件领域的资深技术分析师。
给定产品 {product_name}，你需要：
1. 拆解其物料清单(BOM)，识别5-10个核心零部件
2. 按成本占比排名
3. 标记"标准化(无投资价值)" vs "非标准化(有技术壁垒)"
4. 对每个非标件，判断其技术壁垒等级(极高/高/中/低)
```

## Agent B: 市场集中度审计员

### 触发条件
- 收到Agent A的BOM清单后

### 执行流程
```
对BOM清单中的每个非标准化环节:
1. [搜索] "global [component] market share 2025 CR1 CR3"
2. [搜索] "[component] top suppliers ranking"
3. [提取] 前3大供应商名称、国家、市占率
4. [计算] CR1, CR3, HHI
5. [标记] CR1≥70% → 🔴垄断；CR3≥60% → 🟠寡头
6. [输出] 供应商格局表 → 输入给Agent C
```

### 关键提示词
```
你是全球供应链研究分析师。
给定零部件清单 {component_list}，对每个零部件:
1. 找出全球前5大供应商
2. 标注每个供应商的市占率、所属国家、核心竞争力
3. 计算CR1和CR3
4. 判断是否存在"全球独家"或"寡头垄断"的环节
```

## Agent C: 财务定价权评估员

### 触发条件
- 收到Agent B的供应商格局后

### 执行流程
```
对每个垄断/寡头供应商:
1. [搜索] "[company] gross margin 2025 annual report"
2. [搜索] "[company] R&D ratio patent moat"
3. [搜索] "[company] capacity expansion timeline"
4. [验证] 毛利率 ≥ 50% → 强定价权
5. [验证] 扩产周期 ≥ 12个月 → 供给侧刚性
6. [验证] 研发费用率 ≥ 15% → 持续壁垒
7. [输出] 财务评级 → 输入给Agent D
```

### 关键提示词
```
你是资深买方财务分析师。
给定供应商列表 {supplier_list}，对每个供应商:
1. 查询其毛利率、研发费用率
2. 评估其扩产周期（新产能需要多少个月）
3. 判断其定价权等级(绝对/强/中等/弱)
4. 确认是否满足"卡脖子"财务特征:
   - 毛利率≥50%
   - 扩产周期≥12个月
   - 有专利壁垒
```

## Agent D: 卡脖子综合评级员

### 触发条件
- 收到Agent A/B/C的所有数据后

### 执行流程
```
1. [构建] 使用 build_dependency_graph.py 构建图谱
2. [计算] 介数中心性、PageRank、结构洞
3. [评分] 运行 chokepoint_score() 计算综合评分
4. [评级] 🔴(≥0.70) / 🟠(≥0.50) / 🟡(≥0.30) / 🟢(≥0.15) / ⚪
5. [建议] 生成投资建议和催化剂事件
6. [输出] 完整Chokepoint分析报告
```

### 关键提示词
```
你是投资委员会主席。
综合Agent A/B/C的分析结果:
1. 使用图论方法确认产业链的咽喉要道
2. 给出最终的卡脖子评级
3. 提供可操作的投资建议
4. 标注关键催化剂事件时间表
```

## 工作流编排

### 串行模式（默认）
```python
BOM → 集中度 → 财务 → 评级 → 报告
```

### 并行模式（多个零部件同时查询）
```python
BOM → [并行] 集中度查询1, 集中度查询2, ... 
    → [并行] 财务查询1, 财务查询2, ...
    → 汇总 → 评级 → 报告
```

### 搜索预算控制
- 每轮搜索不超过 5 个 serper 查询
- web_fetch 不超过 3 个页面
- 总数据获取时间控制在 30 秒内
- 超过预算 → 标注"⚠️ 待验证"

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| 搜不到供应商数据 | 标注"⚠️ 待验证"，使用行业常识填充 |
| 搜不到毛利率 | 使用行业平均水平替代，标注"⚠️ 估算" |
| 技术路线不明确 | 列出所有可能路线，说明不确定性 |
| 图谱构建失败 | 降级为纯文本的产业链描述 |
